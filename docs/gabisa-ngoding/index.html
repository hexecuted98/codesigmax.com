Apa Itu Gabisa Ngoding?

Jadi gini bro, Gabisa Ngoding itu bukan cuma nama, tapi juga gaya hidup 😎. Ini tuh bahasa pemrograman parodi dari Brainfuck, tapi versi yang lebih receh dan lokal. Kalau Brainfuck pakai simbol aneh kayak + - < > [ ] . ,, nah Gabisa Ngoding pakai kata-kata yang lebih “manusiawi” kayak puh, sepuh, gabisa, ngoding, dan gengnya.

Gimana Sih Cara Kerjanya?

Oke, sekarang kita bahas dengan serius tapi santai. Jadi bayangin lo punya otak berupa array memori isinya 30.000 slot kosong, semua diisi angka nol.

Terus lo punya satu pointer, alias jarum yang nunjuk ke salah satu slot itu. Nah, setiap perintah di Gabisa Ngoding (alias di Brainfuck) itu mainin jarum dan angka-angka di memori itu.

Perintah Dasar

  • gw → Geser jarum ke kanan (kayak > di Brainfuck)
  • woy → Geser jarum ke kiri (kayak <)
  • gabisa → Tambah angka di memori sekarang (kayak +)
  • ngoding → Kurangin angka di memori sekarang (kayak -)
  • dong → Cetak karakter dari angka di memori sekarang (kayak .)
  • ajarin → Minta input karakter dari user (kayak ,)
  • puh → Mulai loop, alias “selama nilai sekarang gak nol, ulangi bloknya” (kayak [)
  • sepuh → Akhir loop, balik lagi kalau kondisi belum nol (kayak ])

Nih artinya: - Naikin nilai di sel pertama jadi 3 (karena gabisa tiga kali) - Masuk ke loop puh - Geser pointer ke kanan (gw) - Tambah satu (gabisa) - Tutup loop (sepuh) - Cetak hasilnya (dong)

Intinya, kalau Brainfuck itu “komputer yang dikasih makan simbol”, Gabisa Ngoding itu “komputer yang dikasih makan bahasa ngelantur tapi tetep jalan”. Dia gak ngambek, cuma… ya, sering bingung aja :v.

Skema Kerja Otaknya

  1. Program dipecah jadi token (kayak kata puh, gw, dsb)
  2. Token dikompilasi jadi bytecode (angka-angka simpel)
  3. Bytecode dijalankan oleh mesin virtual (VM) yang nyimpen data di array
  4. Setiap perintah ubah pointer, isi memori, atau print karakter

Jadi ya… Gabisa Ngoding ini kayak Brainfuck yang minum es kopi susu — sederhana, overthinking, tapi somehow tetep jalan.
